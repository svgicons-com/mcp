# Design-System Icon Review

This is a documentation example, not a real customer submission.

## Problem

A design-system team wants consistent navigation icons and a reviewable candidate set.

## Solution

Use MCP to compare icon sets, recommend candidates, and optionally create an Icon Collection if the account supports it.

## Tools Used

- `search_icon_sets`
- `recommend_icons_for_ui`
- `search_icons`
- `create_icon_kit`

## Prompts Used

- [Design-system icon review](../../prompts/design-system-icon-review.md)

## Workflow

1. Compare icon sets.
2. Recommend icons for navigation labels.
3. Review style, naming, and accessibility expectations.
4. Create a collection only after review.

## Review Checklist

- Style consistency.
- Naming conventions.
- Provenance notes for team review.
- Permission to create account resources.
