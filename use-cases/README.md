# Community Use Cases

Share what you built with the hosted svgicons.com MCP endpoint.

Good MCP use cases help developers understand a real workflow, useful prompt, setup pattern, or review process. They should be practical, specific, and honest about limitations.

## Example Categories

- AI coding icon search
- React component insertion
- Dashboard navigation
- Design-system review
- Project Kit workflow
- PNG export workflow

## How To Submit

- Open a GitHub issue using the use-case template.
- Or open a pull request adding a Markdown file under `use-cases/`.

## What To Include

- Problem solved
- Stack
- MCP client used
- Tools used
- Prompts used
- Screenshots or GIFs if available
- Optional public repo, demo, or project URL
- Review steps for generated SVG or code

## What Not To Include

- Real API tokens
- Private customer data
- Spam links
- Paid or sponsored claims unless disclosed
- Fake claims
- Private credentials or screenshots

## Link Policy

External links are optional. External links may be reviewed, edited, or removed. User-submitted links may be treated as user-generated content unless editorially reviewed.

The repo prioritizes useful MCP developer workflows.

Start from [TEMPLATE.md](TEMPLATE.md). Sample entries under [examples](examples/) are documentation examples, not real customer submissions.
